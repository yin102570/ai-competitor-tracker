# AI竞品追踪系统 - 启动使用说明

## 一、系统要求

### 必需软件
1. **Node.js** 18+ - 前端运行环境
2. **Python** 3.8+ - 后端运行环境
3. **Redis** - 缓存和任务队列（可选，推荐）

### 可选软件
1. **Docker Desktop** - 容器化部署
2. **Git** - 版本控制

## 二、快速开始

### 方法1：一键启动（推荐新手）
双击运行 `一键启动所有服务.bat`，按照提示操作：

1. **开发模式（推荐）** - 启动所有本地服务
2. **Docker模式** - 使用Docker Compose启动
3. **仅启动后端** - 只启动API服务
4. **仅启动前端** - 只启动前端界面

### 方法2：快速开发
双击运行 `快速开发启动.bat`，自动启动：
- 后端API (端口: 12000)
- Celery Worker (任务处理)
- Celery Beat (定时调度)
- 前端开发服务器 (端口: 12001)

### 方法3：环境检查
首次运行前，建议先运行 `环境检查与安装.bat`：
- 检查所有依赖是否安装
- 自动下载缺失的软件
- 安装项目依赖包

## 三、启动后访问地址

## 五、服务说明

### 1. 后端服务 (Backend)
- **端口**: 12000
- **技术栈**: FastAPI + SQLite + Redis
- **功能**: 提供REST API接口

### 2. 前端服务 (Frontend)
- **端口**: 12001
- **技术栈**: Vue 3 + TypeScript + TailwindCSS
- **功能**: 用户界面和仪表板

### 3. Celery Worker
- **功能**: 异步任务处理（爬虫、数据处理）
- **日志**: 实时显示任务执行情况

### 4. Celery Beat
- **功能**: 定时任务调度
- **日志**: 显示调度执行时间

### 5. Redis
- **端口**: 6379（默认）
- **功能**: 缓存和消息队列

## 六、常见问题解决

### 1. Node.js未安装
运行 `环境检查与安装.bat` 选择自动安装，或手动下载：
https://nodejs.org/zh-cn/download/

### 2. Python未安装
运行 `环境检查与安装.bat` 选择自动安装，或手动下载：
https://www.python.org/downloads/

### 3. Redis启动失败
- 下载Redis Windows版: https://github.com/microsoftarchive/redis/releases
- 或使用Docker: `docker run -d -p 6379:6379 redis:alpine`

### 4. 端口被占用
如果端口12000或12001被占用：
- 修改 `frontend/vite.config.ts` 中的端口号
- 修改后端启动命令中的端口参数

### 5. 依赖安装失败
- 检查网络连接
- 尝试使用国内镜像源：
  ```bash
  # Python镜像
  pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
  
  # npm镜像
  npm config set registry https://registry.npmmirror.com/
  ```

## 七、高级使用

### Docker部署
```bash
# 进入docker目录
cd docker

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 手动启动（命令行）
```bash
# 1. 启动后端
cd backend
venv\Scripts\python -m uvicorn app.main:app --port 12000 --host 127.0.0.1

# 2. 启动Celery Worker
venv\Scripts\python -m celery -A app.workers worker --loglevel=info

# 3. 启动前端
cd frontend
npm run dev
```

## 八、维护与监控

### 查看服务状态
- 后端: http://127.0.0.1:12000/health
- 前端: 检查浏览器控制台
- Redis: `redis-cli ping`

### 日志查看
- 后端API: 查看启动的CMD窗口
- Celery: 查看对应的CMD窗口
- 前端: 查看启动的CMD窗口

### 重启服务
在对应的CMD窗口中按 `Ctrl+C` 停止，然后重新运行启动命令。
